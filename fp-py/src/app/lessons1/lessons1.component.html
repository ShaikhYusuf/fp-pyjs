Lesson 1. What is Functional Programming?

Functional Programming (FP) is a programming paradigm that focuses on writing software by defining and applying functions, without changing states or mutable data. Unlike other programming paradigms, FP avoids altering the program's data, promoting the use of functions to solve problems.
Key Characteristics of FP:
  • Functions are the primary building blocks.
  • Immutability: Once a variable or data structure is created, it cannot be changed.
  • Pure functions: Functions return the same output when given the same input and do not cause side effects.

2. Functional Programming Principles and Advantages

Principles:

  • Pure Functions: The output of a function depends only on its input and does not cause any observable side effects (like modifying a variable outside the function).
  • First-class Functions: Functions can be passed around as values, just like other types (e.g., numbers, strings).
  • Immutability: Data cannot be changed after it’s created. Instead, new data is created from the existing data.

Advantages:

  • Predictable: Pure functions always return the same result for the same input, making your code more predictable.
  • Reusability: Because functions are isolated and don’t rely on external state, they can be reused in many different places.
  • Easier Debugging: Since there are no side effects, you don’t have to track down changes in the program’s state.
  • Testability: Pure functions are easier to test because you don't have to worry about external influences affecting the result.

3. Comparing Functional Programming with Imperative and Object-Oriented Programming in JavaScript and Python

Imperative Programming:

  • Focuses on how to do something, with step-by-step instructions.
  • Uses mutable state and control structures like loops (e.g., for, while).

Object-Oriented Programming (OOP):

  • Focuses on organizing code around objects and classes.
  • Encourages mutating data, which is stored in objects, and using methods to modify that data.

Functional Programming:

  • Focuses on what needs to be done rather than how.
  • Uses pure functions and avoids changing state (immutable data).

4. Functional Programming Paradigms

  • Pure Functions: Functions that produce no side effects and return the same result for the same input.
  • First-class and Higher-order Functions: Functions that can be passed around as arguments or returned as results from other functions.
  • Immutability: Data structures are immutable, meaning once they are created, they cannot be modified.
  • Recursion: Instead of using loops, FP uses recursion to iterate or repeat tasks.
